# Códigos úteis para Git e GitHub

## Windows

- **CD** = Change Directory - modifica o diretório para onde estará indo. Pode ser acrescido de ".." para retroceder o diretório. Acrescentar "/" + nome do diretório para prosseguir.
- **Dir** ou **LS** - Lista todos os diretórios contidos na pasta onde se encontra o bash. Acrescido de "-a", mostra arquivos e pastas ocultas.
- **mkdir** = Make directory - Cria pasta ou arquivo dentro do diretório que você se encontra.
- **del/rmdir** = remove directory - remove arquivo ou pasta no diretório que você se encontra.
- **Ctrl +L** - Limpa a tela dentro do git (também pode ser codado através do código "cls")
- **Tecla Tab** - Autocompleta a pasta (atalho)
- **echo** + texto - Cria um arquivo de texto, caso coloque ".txt"
- **Atalho "seta para cima"** - Reescreve um código que já foi escrito anteriormente. 
- **pwd** - Mostra o "caminho" até a pasta ou diretório em que você se encontra no momento.
- **git init** - Inicia um repositório na máquina local através do gitbash.
- **git add** - Move arquivos e inicia o versionamento. Acrescido de "*'', "." ou "-a" = inclui todos os arquivos para "staged".
- **git commit** - Cria o commit para ser "empurrado" para o github. Acrescido de "-m" + texto, inclui uma mensagem ao ser comitado, informando o que aquele commit está fazendo.
- **git status** - Relata para o usuário o "status" dos arquivos, se os mesmos se encontram modificados ou não ou staged. 
- **mv** - move arquivos para determinada pasta. Ex: mv readme.md ./ DIO = moverá o arquivo readme para a pasta DIO.

### Github ###

-  **git remote add origin + URL ** - Adiciona no github o repositório criado. 
- **git remote -v** - Mostra os repositórios. 
- **git push origin master** - "Leva" o repositório da máquina local para o servidor remoto.
- **git pull origin master** - "Puxa" o repositório do servidor remoto para a máquina local. 
- **git clone + URL** - "Puxa" e "clona" um repositório do github para a máquina local.
